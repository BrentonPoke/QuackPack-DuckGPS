# Contributing to QuackPack Template

Thank you for your interest in contributing to the QuackPack ecosystem! 

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a feature branch (`git checkout -b feature/amazing-feature`)
4. **Make** your changes
5. **Test** thoroughly on actual hardware when possible
6. **Commit** your changes (`git commit -m 'Add amazing feature'`)
7. **Push** to your branch (`git push origin feature/amazing-feature`)
8. **Open** a Pull Request

## Development Guidelines

### Code Style

- Follow Arduino/C++ best practices
- Use clear, descriptive variable and function names
- Comment complex logic
- Keep functions focused and modular

### Testing

- Test on actual hardware when possible
- Document which hardware configurations you've tested
- Include any serial output or logs in PR description

### Documentation

- Update README.md if adding features
- Add comments to complex functions
- Include examples for new functionality

## Creating Your Own QuackPack

When creating a QuackPack from this template:

1. Replace "QuackPackTemplate" throughout the codebase
2. Update `library.json` and `library.properties` with your info
3. Specify supported hardware in README.md
4. Add your specific functionality to the template
5. Create meaningful examples
6. Test on target hardware
